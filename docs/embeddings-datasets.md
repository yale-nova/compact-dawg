# Embedding datasets and binary layout

Index: [AGENTS.md](../AGENTS.md)

This repository's benchmarks operate on large embedding binaries that live under `data/`. The data directory is ignored by Git and should stay that way.

## Main corpus

The current benchmark workflows expect MSMARCO passage embeddings produced with the Qwen3 embedding model family. Lower-dimensional variants are derived from the same source so contributors can compare dimensions without changing the overall data domain.

## Where the data lives

The worktree usually exposes the embedding corpus under:

```text
data/embeddings/qwen3-embedding-0.6b/msmarco_v2/
```

In some environments this path is a symlink into shared storage. The important contract is the directory layout and file naming, not the physical storage location.

## File naming convention

Files follow this pattern:

```text
msmarco_v2_{split}_{dim}d_{dtype}.bin
msmarco_v2_{split}_{dim}d_{dtype}.json
```

Where:

- `{split}` is typically `corpus` or `queries`
- `{dim}` is the embedding dimensionality
- `{dtype}` is `float32` or `float16`

## Binary format

The `.bin` files are flat row-major arrays with no header.

- each vector occupies `dim * element_size` bytes
- vectors are packed back-to-back
- `element_size` is 4 for `float32` and 2 for `float16`

To recover row count from a file:

```text
rows = file_size / (dim * element_size)
```

## Companion metadata

Each binary may have a `.json` sidecar describing dimensionality, dtype, item size, total rows, and any truncation lineage.

## Lower-dimensional variants

Some smaller dimensions are generated by truncating a larger source embedding. The repo helper script for this is:

- [`scripts/truncate_embeddings.py`](../scripts/truncate_embeddings.py)

If a benchmark expects a lower-dimensional file and it is missing, check whether it needs to be derived locally from the larger source.

## Small tracked fixtures

The large benchmark corpora are untracked, but small correctness fixtures under [`tests/testdata/`](../tests/testdata) are committed and safe to use in tests.
